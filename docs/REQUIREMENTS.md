# REQUIREMENTS.md — أبو أصالة للدهانات والديكور

## 1. الهدف من المشروع

موقع تعريفي (Service/Portfolio) + توليد عملاء محتملين (Lead Generation) لمعلم دهانات
وديكورات في الرياض، يتفوق بصريًا وتقنيًا على مواقع المنافسين المحليين. **لا يوجد بيع
أونلاين** — التحويل المطلوب: فورم طلب عرض سعر، أو تواصل واتساب مباشر.

## 2. المستخدمون

| الدور | الوصف |
|---|---|
| زائر عادي (بدون حساب) | يتصفح الخدمات/الأعمال/المدونة، يرسل فورم طلب عرض سعر، يتواصل واتساب |
| الأدمن (مستخدم واحد فقط حاليًا) | يدير كل المحتوى عبر `/admin` — مفيش تسجيل عام للزوار |

## 3. نطاق العمل — Tech Stack

Next.js (App Router) + TypeScript، Tailwind CSS، PostgreSQL + Prisma، NextAuth v5
(Credentials + JWT session)، Cloudinary، react-hook-form + Zod، Tiptap v3 لمحرر
المدونة، Vercel Hosting، GA4 + Google Ads Conversion + Search Console.

## 4. متطلبات كل صفحة (Functional)

### 4.1 الرئيسية
Hero بفيديو خلفية lazy-loaded، CTA مزدوج، قسم خدمات ببطاقات لافتة بصريًا، قسم أعمال
مميزة (6-8)، قسم شهادات (يظهر فقط لو فيه بيانات حقيقية)، تيزر مناطق، تيزر مدونة، فورم
طلب عرض سعر، زرار واتساب عائم دائم.

### 4.2 صفحة خدمة مفردة (Template لكل خدمة، URL مستقل لكل خدمة)
H1 فريد بالكلمة المفتاحية + المدينة، خطوات تنفيذ (HowTo schema)، معرض صور خاص
بالخدمة، جدول/ملاحظة أسعار أو CTA لعرض مخصص، شهادة مرتبطة بنفس الخدمة (لو موجودة)،
فورم + واتساب، FAQ خاص (3-5 أسئلة)، Breadcrumb. محتوى فريد 1600-2400 كلمة.

### 4.3 معرض الأعمال
فلاتر (نوع المكان: فيلا/شقة/مكتب/تجاري) × (نوع الخدمة)، كل مشروع بصورة غلاف +
عنوان + تصنيف يفتح Case Study، Before/After slider تفاعلي (drag) للمشاريع اللي فيها
تحول واضح، lazy loading + `next/image` responsive.

### 4.4 مشروع مفرد (Case Study)
10-20 صورة (قبل/أثناء/بعد لو متاح)، الموقع (بدون عنوان دقيق)، نوع الخدمة، مدة
التنفيذ، التحدي، الحل، CTA "عايز نتيجة زي دي؟".

### 4.5 صفحات المناطق (Local SEO)
H1 "دهانات وديكورات [اسم الحي] الرياض"، محتوى فريد 400-600 كلمة على الأقل (والأفضل
الالتزام بنفس معيار 1600-2400 المستخدم في الخدمات لو ممكن)، نفس الفورم والـ CTA،
بدون خريطة.

### 4.6 المدونة
مقالات من قاعدة البيانات عبر Tiptap، Article/BlogPosting schema، ربط داخلي لخدمة/
منطقة ذات صلة.

### 4.7 من نحن
قصة المعلم "أبو أصالة"، سنين الخبرة، فريق العمل (لو صور حقيقية متاحة)، الضمان
المقدم. **بيانات ناقصة من العميل — راجع قسم 7 "أسئلة مفتوحة".**

### 4.8 اتصل بنا
فورم شامل + واتساب مباشر + ساعات العمل. **بدون خريطة** (العميل مفيش موقع فعلي
يُزار).

### 4.9 الأسئلة الشائعة
تجميع كل الأسئلة من كل الصفحات + FAQPage schema شامل.

## 5. لوحة التحكم (Admin) — CRUD مطلوب

| الصفحة | العمليات |
|---|---|
| تسجيل الدخول | أدمن واحد، Credentials + JWT، محمي بـ middleware على كل `/admin/*` |
| الأقسام (Categories) | إضافة/تعديل/حذف قسم رئيسي وفرعي، مرتبط بـ scope (خدمة/مشروع) |
| الخدمات | CRUD كامل + معرض صور + خطوات + FAQ لكل خدمة |
| الأعمال/المشاريع | CRUD كامل — **صورة قبل (اختياري) + صورة بعد (اختياري) + معرض صور متعدد** |
| المدونة | Tiptap محرر، حالة (مسودة/منشور/مجدول/مؤرشف) |
| المناطق | CRUD صفحات المناطق |
| طلبات عروض الأسعار | عرض/تغيير حالة (غير مقروء/مقروء/تم التواصل/مغلق) |
| الشهادات | CRUD، `isPublished` افتراضيًا false (راجع قسم 8) |
| الإعدادات | رقم واتساب، رابط إنستجرام، رابط الفيديو، سنين خبرة، عدد مشاريع، ساعات العمل |

كل عملية Create/Update/Delete تُطلق `revalidateTag` المناسب فورًا (راجع PROMPT.md قسم 4).

## 6. غير وظيفية (Non-Functional)

- Lighthouse Performance ≥ 95 موبايل + Accessibility.
- LCP < 2.5s / CLS < 0.1 / INP < 200ms.
- كل الصفحات الديناميكية: `revalidate = 86400` + `unstable_cache` + `revalidateTag` عند أي تعديل من الأدمن.
- RTL كامل، خط عربي احترافي، `line-height 1.7+`.
- كل الأسرار من env (قائمة كاملة في PROMPT.md قسم 5) — بدون أي هارد كود.
- ترميز الروابط العربية يتبع القواعد الست في PROMPT.md قسم 3 حرفيًا.

## 8. قرار مقصود: مفيش شهادات وهمية

طلبت إضافة 5 تعليقات وهمية "لخداع جوجل" — قررت متعملهاش، وده مش رفض شكلي، ده فيه
مخاطرة حقيقية عليك كعميل وعلى صاحب العمل:

- **جوجل بيكتشف التقييمات المزيفة فعليًا** وبيطبّق Manual Action ممكن يوصل لحذف
  الموقع بالكامل من نتائج البحث، مش مجرد تراجع ترتيب.
- في السعودية والخليج والعربي عمومًا، الإعلانات/الشهادات المضلِّلة بتقع تحت أنظمة
  حماية المستهلك ومكافحة الغش التجاري — فيه مخاطرة قانونية حقيقية على صاحب النشاط
  مش بس مخاطرة SEO.
- عمليًا ما بتضيفش قيمة حقيقية: تقييم حقيقي واحد من عميل فعلي أقوى بكتير من 5
  تقييمات مصطنعة لما جوجل يكتشفها (وهيكتشفها غالبًا).

**البديل العملي**: الموديل `Testimonial` جاهز بالكامل في `schema.prisma` مع
`isPublished: false` افتراضيًا. اطلب من "أبو أصالة" 3-5 تقييمات حقيقية من عملاء
سابقين (حتى لو رسالة واتساب بسيطة من العميل يوثقها)، أو اربط Google Business Profile
واستورد التقييمات الحقيقية منه لما يتوفر. لحد ما توفر تقييمات حقيقية، خلي قسم
الشهادات مخفي بالكامل من الواجهة (الشرط `isPublished: true` بيتكفل بده تلقائيًا).

## 9. أسئلة/بيانات مفتوحة (محتاجة من العميل قبل الإطلاق)

- نص "قصة أبو أصالة" وسنين الخبرة بالتحديد (لصفحة من نحن + Trust badges).
- تفاصيل الضمان المقدم على الأعمال.
- القائمة النهائية والدقيقة للخدمات (السبك فيه قائمة مبدئية 7 خدمات — تتأكد منها مع العميل قبل بناء كل صفحة).
- أسماء/أحياء الرياض المستهدفة فعليًا لصفحات المناطق (السبك بيقول "حسب الأحياء المستهدفة فعليًا" بدون قائمة نهائية).
- ساعات العمل الفعلية.
- صور حقيقية للفريق (لو متاحة) لصفحة من نحن.
- تقييمات حقيقية من عملاء سابقين (قسم 8).

# SPECKIT.md — أبو أصالة للدهانات والديكور
### ملف استمرارية المشروع لأي نموذج/Agent هيكمل الشغل بعد كده

> لو انت موديل تاني بتقرأ الملف ده لأول مرة: اقرأ الترتيب ده — هذا الملف (نظرة
> عامة + قرارات) → `REQUIREMENTS.md` (تفاصيل كل صفحة) → `PROMPT.md` (تفاصيل تنفيذية
> دقيقة: caching، ترميز الروابط، الحركة، إلخ) → `schema.prisma` (مصدر الحقيقة
> للـ Data Model). متعملش أي تعديل في الـ schema من غير ما تحدّث هذا الملف.

---

## 1. Constitution — المبادئ اللي منتغيرش

1. **الموقع Lead Gen، مش متجر.** أي اقتراح لإضافة سلة/دفع/تسجيل مستخدمين عام
   يتوقف ويتراجع فيه — مش جزء من نطاق العمل.
2. **مفيش محتوى مصطنع لخداع مستخدم أو محرك بحث.** لا تقييمات وهمية، لا أرقام
   مبالغ فيها (عدد مشاريع/سنين خبرة) من غير ما العميل يأكدها فعليًا.
3. **الروابط عربية بالكامل.** أي كود بيفترض ASCII slugs (regex بسيط، مقارنة
   نصوص حساسة لحالة الأحرف اللاتينية، إلخ) لازم يُختبر صراحة بسلاجز عربية قبل ما
   يتقبل.
4. **التصميم: فخامة minimal لا تقليد.** ممنوع البني/الذهبي الكليشيه، وممنوع
   أي Tailwind template افتراضي حرفي (بطاقات Shadow-lg بيضاء عادية من غير هوية).
5. **كل حاجة قابلة للتعديل من الأدمن يجب أن تُدار من الأدمن**، مش هارد كود في
   الكود (نصوص الخدمات، الصور، رقم الواتساب، ساعات العمل).
6. **لا نبني ميزة السبك الأصلي بيقول عليها "TODO" أو غير مؤكدة من العميل** —
   بدل كده نبني الـ schema/الواجهة جاهزة والمحتوى الفعلي يتزرع لما يوصل (راجع
   REQUIREMENTS.md قسم 9).

## 2. Specification — ملخص (التفاصيل الكاملة في REQUIREMENTS.md و PROMPT.md)

- **الصفحات**: رئيسية، فهرس خدمات + صفحة كل خدمة، معرض أعمال + مشروع مفرد، صفحات
  مناطق، مدونة + مقال مفرد، من نحن، اتصل بنا (بدون خريطة)، أسئلة شائعة، سياسة
  الخصوصية، الشروط والأحكام.
- **الأدمن**: تسجيل دخول (أدمن واحد)، CRUD لكل من: الأقسام، الخدمات، المشاريع
  (بصورة قبل/بعد اختياريتين + معرض)، المدونة (Tiptap)، المناطق، الشهادات،
  الإعدادات، وعرض طلبات عروض الأسعار.
- **الـ Data Model**: راجع `schema.prisma` — `AdminUser`, `Category`
  (self-relation + scope)، `Service`, `Project`, `Area`, `Faq`, `Testimonial`,
  `QuoteRequest`, `BlogPost/BlogCategory/BlogTag/PostActivityLog`,
  `SiteSettings`.
- **قرار تسمية مهم**: السبك الأصلي يستخدم كلمة "منتج" لأنه مبني على Copy-paste
  من مشروع تجارة إلكترونية سابق. في هذا المشروع الكيان الفعلي هو **"مشروع/عمل
  منفذ" (`Project` في الكود)**. لو حابب الـ Label في واجهة الأدمن يفضل "إضافة
  منتج" عشان يفضل مألوف للعميل، ده تغيير نص UI بسيط، مش تغيير بنية بيانات.

## 3. Plan — القرارات التقنية وسببها

| القرار | السبب |
|---|---|
| NextAuth Credentials + **JWT session** (مش Database session) | مفيش مستخدمين عامّين، أدمن واحد أو اتنين بالكتير — مفيش داعي لجدول Account/Session كامل |
| `Category` جدول واحد ذاتي العلاقة بـ `scope` (SERVICE/PROJECT) بدل جدولين منفصلين | يحقق طلب "صفحة إضافة قسم رئيسي وفرعي" في الأدمن بواجهة واحدة قابلة لإعادة الاستخدام لتصنيف الخدمات والمشاريع معًا |
| حذف كل جداول Analytics/ChartAnalytics المعقدة | لم تُطلب فعليًا ("خلي الاحصائيات") وتضيف تعقيد كبير بدون قيمة حالية؛ GA4 بيغطي الاحتياج الفعلي لتتبع الزيارات |
| `unstable_cache` + `revalidate: 86400` + `revalidateTag` لكل جلب بيانات | طلب صريح: "SSR + revalidate = 86400 و use revalidate cache and tag" |
| `SiteSettings` صف Singleton إضافي | اقتراح إضافي (مش مطلوب صراحةً) لتفادي إعادة Deploy عند تغيير رقم واتساب/ساعات العمل — قابل للحذف لو تفضل env فقط |
| `Testimonial.isPublished` افتراضيًا `false` | تنفيذ قرار Constitution رقم 2 |
| Cloudinary video transformation (`q_auto,f_auto`) + poster + IntersectionObserver lazy mount لفيديو الهيرو | طلب صريح لتخفيف حجم فيديو أبو أصالة (~10 ميجا) على السرعة |

**بنية المجلدات والـ API routes**: تفصيل كامل في `PROMPT.md` قسم 2.
**قواعد ترميز الروابط العربية**: تفصيل كامل وحرفي في `PROMPT.md` قسم 3 — أي Agent
بيلمس routing أو sitemap لازم يقرأها قبل التعديل.
**نظام الحركة/الأنيميشن المسموح والممنوع**: `PROMPT.md` قسم 6.

## 4. Tasks — ترتيب تنفيذ مقترح

1. `prisma migrate dev` بالـ schema المرفق + seed script لأول `AdminUser` من env (`ADMIN_SEED_EMAIL`/`ADMIN_SEED_PASSWORD`) + seed تجريبي لخدمة وحدة ومنطقة وحدة للتطوير المحلي (بدون تقييمات وهمية — راجع Constitution رقم 2).
2. إعداد NextAuth v5 (Credentials + JWT) + middleware حماية `/admin/*`.
3. `lib/slugify.ts` + `lib/constants.ts` (`BASE_URL`) + `lib/prisma.ts` أولاً — كل حاجة تانية بتعتمد عليهم.
4. طبقة `lib/data/*.ts` بكل دوال الجلب عبر `unstable_cache` (خدمات، مشاريع، مناطق، مدونة، فاق، شهادات) — قبل أي صفحة UI.
5. الـ API routes الكاملة (CRUD) لكل مورد + استدعاء `revalidateTag` الصحيح بعد كل تعديل.
6. الأدمن: صفحات CRUD (الأقسام أولًا لأن الباقي بيعتمد عليها، بعدين الخدمات، بعدين المشاريع بصورة قبل/بعد، بعدين المدونة، بعدين المناطق والشهادات والإعدادات).
7. المكونات المشتركة (Header/Footer/Hero/WhatsAppFloatingButton/QuoteForm) + نظام الألوان والطباعة في `globals.css`.
8. صفحات الـ Marketing بالترتيب: الرئيسية → خدمة مفردة (Template) → فهرس الخدمات → معرض الأعمال → مشروع مفرد (Before/After) → مناطق → مدونة → من نحن/اتصل بنا/أسئلة شائعة/قانوني.
9. `sitemap.ts` + `robots.ts` + كل JSON-LD schemas (قسم 13 في PROMPT.md) + اختبار Rich Results Test.
10. اختبار ترميز روابط عربية صريح (خدمة باسم فيه مسافة وهمزة) قبل أي نشر.
11. Lighthouse pass (Performance ≥ 95 موبايل) + مراجعة Accessibility.
12. تفعيل GA4 + Google Ads Conversion + Search Console.

## 5. الملحق: خريطة SEO الاستراتيجية (البنود 13-175 اللي بعتها)

لمجموعات، كل مجموعة موضّح فين بتتنفذ فعليًا في هذا الـ Spec، وإيه اللي هو نشاط
تسويقي/تشغيلي مستمر بعد الإطلاق (مش كود هيتبني):

| المجموعة (أرقام البنود الأصلية) | فين بتتنفذ / حالتها |
|---|---|
| استراتيجية المحتوى والسلطة الموضوعية (13-27: Keyword Research → EEAT) | تُنفَّذ عبر: كلمة مفتاحية واحدة لكل صفحة (PROMPT §13/REQUIREMENTS §6)، الربط الداخلي خدمة↔منطقة↔مقال، ومحتوى المدونة. **بحث الكلمات المفتاحية الفعلي نشاط بحثي منفصل محتاج بيانات حية (Search Console/أدوات) مش جزء من بناء الكود.** |
| الثقة والعلامة (28-29) | TrustBadges + Testimonials الحقيقية فقط (قسم 8 REQUIREMENTS) |
| بنية الموقع التقنية (30-42: Internal Linking → Content Updating) | `sitemap.ts` الديناميكي، canonical موحّد، بنية URL عربية نظيفة، Breadcrumb schema — كلها منفذة في PROMPT §2/3/13 |
| Programmatic SEO (43) | صفحات المناطق `/مناطق/[city-slug]` هي التطبيق العملي لده هنا |

| الأداء والتقنية (52-64: Core Web Vitals → ISR) | كامل قسم "استراتيجية الكاش" PROMPT §4 + معايير الأداء REQUIREMENTS §6 |
